## Типы участников проекта
Существует три типа участников проектов:
* Пользователи системы Все зарегистрированные пользователи системы могут быть назначены участником любого из проектов. Каждый пользователь назначается в проект с конкретным набором ролей (например, администратор, участник, читатель и так далее)
* Группы пользователей Пользователей можно объединять в группы и разом назначить их в проект. В этом случае всем пользователям группы назначается один набор ролей.
* Пользователи-заполнители Это несуществующие пользователи. С ними удобно работать на этапе, когда еще не известно, кто конкретно будет выполнять задачу, но уже четко определена роль (например, разработчик, тестировщик, аналитик).

## Каких пользователей можно пригласить в проект
*   новых пользователей, [пригласив](/o/DqjJoZdtLJDvqwHUB8fg/s/iGbgSLUDIqxrq3NUAfL9/~/diff/~/changes/65/rukovodstvo-polzovatelya/rabota-s-uchastnikami#dobavlenie-uchastnikov) их по email; В этом случае происходит регистрация пользователя и одновременное добавление в проект с указанными ролями.
*   существующих в системе пользователей, выбрав их из списка доступных; У вас могут быть пользователи, которые являются участниками других проектов или не состоящие в проектах. Здесь вы можете выбрать их и пригласить в проект.

## Группы пользователей в качестве участников проекта
Группа — это список пользователей, которых можно назначить на проект с определённой ролью. Группы оказывают следующее влияние на список участников проекта:

*   Все участники группы автоматически получают права, соответствующие ролям, присвоенным при группе.
*   При добавлении нового пользователя в группу, которая уже назначена на какие-либо проекты, он сразу получает права на проект в соответствии с ролями группы.
*   При удалении пользователя из группы, права, предоставленные через эту группу, будут автоматически отозваны.
*   Группа отображается отдельной строкой в списке участников проекта. Также для групп отображаются все её участники.
*   Участники группы не могут быть удалены из списка участников проекта по отдельности.
*   Если в проект уже добавлен участник, который является членом группы, то роли группы и пользователя будут суммироваться.
*   Пользователю можно присвоить дополнительные роли, которые не зависят от роли группы, в которой он состоит.
*   Роль, назначенная группе, не может быть изменена для отдельных участников внутри этой группы.

Механизм группового управления упрощает управление правами и доступом к проекту, обеспечивая гибкость и единообразие настройки ролей.

<Tip>Рекомендуется использовать группы в качестве участников проекта, если у вас большая компания или команда. Так будет легче администрировать добавление и удаление пользователей. Вы один раз добавляете / удаляете пользователя, и он автоматически получает права группы во всех проектах, куда добавлена эта группа.</Tip>

## Пользователи-заполнители в качестве участников проекта

Пользователи-заполнители —это виртуальные пользователи, которые не имеют доступа к проекту. Они создаются без адреса электронной почты (им не отправляются письма и уведомления), на низ не расходуются лицензии.

Назначение пользователей-заполнителей — это планирование проекта на ранних этапах. Благодаря пользователям-заполнителям вы заранее сможете определить проектные роли и потребность в ресурсах. В последствии, когда будут известны конкретные участники проекта производится замена пользователей заполнителей в задачах на реальных пользователей.

<Tip>
Рекомендуется использовать пользователей-заполнителей когда команда еще не укомплектована. Например, вы уже знаете, что задачу будет выполнять Дизайнер, но кто именно пока не известно. В этом случае вы можете добавить пользователя-заполнителя "Дизайнер", спланировать задачи для него. Позже вы его сможете групповой заменой изменить его, к примеру, на "Василия Петрова".
</Tip>

Пользователь-заполнитель в списке участников будет выглядеть немного иначе, чем обычные пользователи:
* будет показан без адреса электронной почты;
* не может быть добавлен в группу, т.е. будет показаны без принадлежности к группе.
